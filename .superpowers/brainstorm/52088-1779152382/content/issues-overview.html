Audit App — Issues Found

Here's what I spotted across audit_app.py, admin.py, and ui/styles.py

🔴 Consistency problems
Inline styles in headers

audit_app.py and admin.py both hardcode font-family:Georgia,serif;color:#1e3a5f inline instead of using the CSS token variables from SHARED_CSS. If you update --accent in styles.py, the headers won't follow.

Admin page has no CSS classes

admin.py hard-codes every color and font as inline styles (~30 occurrences). SHARED_CSS defines tokens and classes that admin doesn't use at all — making the two pages visually inconsistent at the detail level.

🟠 Labeling UX issues
Label buttons are colorless

BINDING is navy (primary), but RELEVANT and IRREL. are plain gray. The CSS already has --binding: #166534, --relevant: #92400e, --irrelevant: #991b1b — none of which reach the buttons. Labeling is the core action of the app, so these should be visually distinct.

"IRREL." is truncated

The third label button reads "IRREL." — truncated awkwardly. This is the full width of the button in the tight 3-column layout. Either fix the truncation or rethink the layout so all three labels fit properly.

No persistent saved-label highlight on card

After labeling, the expander header gets a small emoji icon, but the card itself has no color change. When reviewing many chunks, it's hard to tell at a glance which have been labeled and which haven't.

History list buttons are awkward

The history panel uses st.button() with \n in the label for a two-line layout. Streamlit doesn't render this consistently — the result looks rough and cramped. Label/date stats should be rendered separately from the query text.

🔵 Minor polish gaps
st.write("") vertical-align hacks

Both files use st.write("") as a spacer to push buttons down. This creates invisible gaps and is fragile across Streamlit versions.

Login/Signup forms are full-width

The login and signup forms stretch the full page width. SHARED_CSS already defines an .ejp-card class (max-width 480px, centered) but neither form uses it, so they look very wide on large screens.

Sidebar is minimal

Shows email + Admin Panel button. The Admin Panel button is visible to all logged-in users, not just admins. The sidebar also has no back-to-audit-app button when you're on the admin page.
